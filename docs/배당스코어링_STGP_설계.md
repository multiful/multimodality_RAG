# 배당 스코어링 테이블(STGP) — KOSPI200 적용 설계

> 논문 「배당 스코어링 테이블을 활용한 정량적 투자 전략」(양지원·조재일·유재필,
> JKAIS Vol.25 No.12, pp.411-419, 2024)의 STGP(Scoring Table for Growth Potential)를
> **KOSPI200 개별 종목**에 적용해 종목별 배당 스코어를 산출하고,
> 그 결과를 **RAG DB(Supabase)** 에 적재하기 위한 설계 문서.
>
> 산출물 짝: `supabase/migrations/002_dividend_score.sql` (테이블·RPC DDL)
> 작성일: 2026-07-24 · 스코어 버전: v1(논문 원식) / v2(고도화)

---

## 0. 요약 (TL;DR)

- **v1**: 논문 Table 1을 그대로 구현. 7개 항목(x11~x42) 합산 → GP ∈ [-4, +4] → A/B/C 그룹.
- **v2**: v1의 계단식 점수를 **백분위·연속 점수로 정교화**하고, **범주 5(배당 안정성·여력:
  FCF 커버리지, 연속 지급/증가 연수, 수익률 밴드)** 를 추가. GP_v2 ∈ [-5, +5].
- **핵심 데이터 소스**: 배당성향·주당배당금·현금배당총액·당기순이익은 **OpenDART
  `alotMatter`(배당에 관한 사항) API 한 방**으로 당기/전기/전전기 3개년이 나온다.
  자기자본·현금흐름은 `fnlttSinglAcntAll`, 시세성 지표(DIV·DPS)와 KOSPI200 구성종목은
  `pykrx`. 검증·보조는 FnGuide 스냅샷.
- **RAG 적재**: 원시값은 `dividend_facts`, 점수는 `dividend_scores`에 정형 저장하고,
  종목별 **한국어 요약문(embed_text)을 BGE-M3로 임베딩**해 기존 하이브리드 검색과
  같은 패턴(`match_dividend_scores` RPC)으로 검색 가능하게 한다.

## 1. 논문 원 산식 (v1 — 그대로 유지)

### 1.1 STGP 스코어링 테이블 (논문 Table 1 재현)

| 범주 | 항목 | 정의 | 채점 기준 | 점수 |
|---|---|---|---|---|
| 1. 지급 이력 | **x11** | 과거 3년간 배당 지급 횟수 | 3회 | +0.5 |
| | | | 1~2회 | 0 |
| | | | 0회 | -0.5 |
| | **x12** | X = 배당 지급 횟수 ÷ 순이익 흑자 횟수 (과거 3년) | X = 1 | +0.5 |
| | | | 1 > X > 1/3 | 0 |
| | | | 1/3 ≥ X ≥ 0 | -0.5 |
| 2. 배당성향 | **x21** | 배당성향(= 배당금총액 ÷ 당기순이익 × 100)의 증감 | 증가 | +0.5 |
| | | | 감소 | 0 |
| | **x22** | 순이익 변동 vs 배당성향 변동 | Δ순이익 > Δ배당성향 | +0.5 |
| | | | Δ순이익 < Δ배당성향 | -1 |
| 3. 현금배당률 추세 | **x3** | 현금배당률(= 현금배당 ÷ 자기자본)의 3개년 추세 (가중 합산) | t vs t-1 증가 / 감소 | +0.5 / -0.5 |
| | | | t-1 vs t-2 증가 / 감소 | +0.3 / -0.3 |
| | | | t-2 vs t-3 증가 / 감소 | +0.2 / -0.2 |
| 4. 상대 위치 | **x41** | 배당성향의 유니버스 내 상대 위치 | 하위 30% (여력 큼) | +0.5 |
| | | | 상위 30% (여력 소진) | -0.5 |
| | **x42** | ROE의 유니버스 내 상대 위치 | 상위 30% | +0.5 |
| | | | 하위 30% | -0.5 |

- 총점: **GP = x1 + x2 + x3 + x4** (x1 = x11+x12, x2 = x21+x22, x4 = x41+x42)
- 점수 범위: **-4 ≤ GP ≤ +4**
  - 최대 = 0.5+0.5 +0.5+0.5 +(0.5+0.3+0.2) +0.5+0.5 = **+4.0**
  - 최소 = -0.5-0.5 +0-1 -(0.5+0.3+0.2) -0.5-0.5 = **-4.0**
- 그룹핑: **A: +1 ≤ GP ≤ +4 · B: -1 ≤ GP < +1 · C: -4 ≤ GP < -1**
- 최신 시점(t) 변화에 최근 가중(0.5 > 0.3 > 0.2)을 두는 것이 설계 의도.
  x22의 -1점은 "이익이 못 따라가는 배당 확대(무리한 배당)"에 대한 강한 페널티.

### 1.2 논문 실험 결과 (참고)

- 대상: 나스닥 배당주 500종목, 2020~2023, 연 1회 리밸런싱.
- STGP 기반 포트폴리오(PM-GP)가 DEA 기반(PM-DEA)·벤치마크 대비 대체로 우수
  (예: 2023년 A/B그룹 71%/78% vs BM 54%). 하락장(2022)에서도 상대 방어.
- 시사점: **A그룹(GP ≥ +1) 종목 수가 안정적으로 유지**되는 것 자체가 지표의 안정성 근거.

### 1.3 KOSPI200 적용 시 조정 사항

| 항목 | 논문(나스닥) | KOSPI200 적용 |
|---|---|---|
| 유니버스 | 나스닥 배당주 500 | KOSPI200 구성종목 (pykrx 지수코드 `1028`) |
| 상대평가 모집단 (x41·x42의 30% 컷) | 500종목 전체 | **KOSPI200 내 백분위**로 계산 (섹터 옵션은 v2 참고) |
| 회계 기준 | 미국 회계연도 | **연결(CFS) 우선**, 없으면 별도(OFS). 12월 결산 기준, 3월 결산 등 예외는 직전 확정 사업연도로 정렬 |
| 배당 데이터 | Google API | OpenDART + pykrx (아래 §3) |
| 리밸런싱 | 연초 구성 → 연말 평가 | 사업보고서 공시 확정 후 **매년 4월 갱신** 권장 (12월 결산법인 보고서 제출기한 3/31) |
| 주의 | — | 분기·중간배당(삼성전자 등)은 연간 합산 DPS 기준으로 통일. 우선주는 보통주와 분리, 스코어는 보통주 기준 |

## 2. 고도화 산식 (v2)

v1은 그대로 두고(재현성·논문 대조용), v2를 별도 버전으로 병행 산출한다.
`dividend_scores.score_version` 컬럼('v1' | 'v2')으로 구분 — 같은 종목·연도에 두 버전 공존.

### 2.1 정교화 ① — 계단식 → 연속 점수

계단식(±0.5)은 경계 근처 종목의 점수가 미세한 값 차이로 튀는 문제(불연속)가 있다.
v2에서는 각 항목을 같은 범위 안의 **연속 점수**로 바꾼다. 원칙:

- **상대평가 항목(x41, x42)** — 30% 컷 대신 백분위 연속화:
  - `x41' = clip(0.5 - pr(payout), -0.5, +0.5)` … `pr` = 유니버스 내 백분위(0~1). 배당성향이 낮을수록 +.
  - `x42' = clip(pr(ROE) - 0.5, -0.5, +0.5)` … ROE가 높을수록 +.
  - 백분위 계산 전 **윈저라이즈(1%, 99%)** 로 극단치 완화.
- **증감 항목(x21, x3)** — 부호 판정 대신 변화율 스케일링:
  - `x21' = 0.5 × clip(Δpayout / B21, -1, +1)`, 밴드 `B21 = 10%p` (10%p 증가면 만점).
    단, v1과 달리 **감소도 음수**(-0.5까지)로 반영 — v1의 "감소=0"은 정보 손실.
  - `x3' = Σ_k w_k × clip(Δcashdiv_k / B3, -1, +1)`, `w = (0.5, 0.3, 0.2)`, `B3 = 0.5%p`
    (현금배당률은 자기자본 대비라 변동폭이 작으므로 밴드를 좁게).
- **x22 (무리한 배당 페널티)** — 비대칭 구조 유지하되 연속화:
  - `g = Δ순이익율 - Δ배당성향율` (전년 대비 변화율 차).
  - `x22' = +0.5 × clip(g / 0.2, 0, 1)` (g > 0일 때), `-1.0 × clip(-g / 0.2, 0, 1)` (g < 0일 때).
- **x11, x12** — 이산 특성이 본질이므로 v1 그대로 유지.
- **범주별 클립**: v2에서는 각 범주 합을 **[-1, +1]로 클립**한다
  (`x2' = clip(x21' + x22', -1, +1)` 등). x21을 음수 허용으로 바꾸면서 범주2 하한이
  -1.5까지 내려가는 것을 막아, 다섯 범주가 동일 범위(±1)로 총점에 기여하게 한다.

### 2.2 정교화 ② — 결측·특수상황 규칙 (실무에서 가장 중요)

| 상황 | 규칙 |
|---|---|
| 순이익 적자 연도 | 배당성향 정의 불가 → 해당 연도 x21·x22·x41은 **0점 처리** + `flags`에 `deficit_year` 기록. x12의 "흑자 횟수" 분모에는 정상 반영 |
| 3년 미만 상장(데이터 부족) | 있는 연도만으로 계산하되 `coverage_years < 3`이면 `flags: short_history`, x11은 있는 연도 기준 비율로 환산하지 않고 **결측 연도 = 미지급으로 간주하지 않음** → x11 = 0점 고정 |
| 배당성향 > 200% 또는 < 0 | 200%로 캡 / 0 처리 + `flags: payout_outlier` (일회성 손익 왜곡) |
| 특별배당 감지 (DPS가 전년 대비 3배↑ 후 복귀) | `flags: special_dividend` — x3 추세 계산에서 해당 연도 스무딩(전후 평균 대체) 옵션 |
| 지주회사·금융업 | FCF 정의가 다름(§2.3 x51) → 금융업은 x51 = 0점 + `flags: financial_sector` |
| 무배당 지속 기업 | GP 산출은 하되 `flags: non_payer` — RAG 요약문에 명시 |
| 필수 원천값 자체 결측 | 해당 항목 0점 + `data_quality` JSONB에 결측 필드 기록. **결측을 음수로 처리하지 않는다** (수집 실패와 펀더멘털 악화를 구분) |

### 2.3 지표 추가 — 범주 5: 배당 안정성·여력 (x5, 범위 [-1, +1])

논문에 없는 보조지표. 배당의 **지속 가능성**을 직접 본다.

| 항목 | 정의 | 채점 | 점수 |
|---|---|---|---|
| **x51** FCF 배당커버리지 | FCF(영업활동현금흐름 - CAPEX) ÷ 현금배당금총액 | ≥ 2.0 | +0.5 |
| | | 1.0 ~ 2.0 | +0.25 |
| | | < 1.0 (FCF로 배당 못 덮음) | -0.5 |
| **x52** 배당 연속성 | 연속 지급 연수 ≥ 5년 | | +0.25 |
| | 연속 증가(DPS) 연수 ≥ 3년 | (지급과 중복 가산, 최대 +0.5) | +0.25 |
| | 직전 연도 감배 또는 무배 전환 | (가산 무시하고 단독 적용) | -0.5 |
| **x53** 수익률 밴드 (밸류에이션 보정) | 현재 배당수익률 vs 자사 5년 평균 | 평균 +1σ 이상 (역사적 저평가) | +0.25 |
| | | 평균 -1σ 이하 (역사적 고평가) | -0.25 |
| | 밸류트랩 경고: 수익률 +2σ 이상 **이면서** x51 < 1 | (위 점수 대신 단독 적용) | -0.5 |

- x5 = x51 + x52 + x53 이되 **[-1, +1]로 클립**.
- x52는 5~10년 이력 필요 → v2는 **최소 5개년, 권장 10개년** 수집 (v1은 4개년이면 충분: t-3~t).

### 2.4 v2 총점·그룹

- **GP_v2 = x1 + x2' + x3' + x4' + x5 ∈ [-5, +5]** (각 범주 [-1, +1] 클립 — §2.1)
- 그룹 경계는 v1의 ±1을 범위비(5/4)로 확장: **A: GP_v2 ≥ +1.25 · B: -1.25 ≤ GP_v2 < +1.25 · C: < -1.25**
- 표시용 정규화: `score100 = (GP_v2 + 5) / 10 × 100` (0~100). v1도 동일식으로 (GP+4)/8×100.
- 유니버스 내 순위(`rank_in_universe`, `pct_in_universe`)를 함께 저장 — RAG 답변에서
  "KOSPI200 내 상위 n%"로 바로 인용.
- (옵션, 후순위) 섹터 상대평가: x41·x42·x53의 백분위를 GICS 섹터 내로 좁히는
  `universe='KOSPI200_SECTOR'` 변형. 금융·유틸리티(고배당성향 정상)와 성장주 왜곡을
  줄이지만, 섹터당 표본이 얇아지므로 **종목 수 ≥ 15인 섹터만** 적용.

## 3. 데이터 수집처 — 지표별 소스 매핑 ★

### 3.1 소스 총람

| 소스 | 무엇을 | 방법 | 비용/제약 |
|---|---|---|---|
| **OpenDART `alotMatter`** (배당에 관한 사항) | 주당 현금배당금, **현금배당성향(%)**, 현금배당수익률(%), 현금배당금총액, (연결)당기순이익, EPS — **당기/전기/전전기 3개년이 한 호출에** | `GET https://opendart.fss.or.kr/api/alotMatter.json?crtfc_key=…&corp_code=…&bsns_year=YYYY&reprt_code=11011` 또는 `OpenDartReader.report(ticker, '배당', year)` | 무료 키, **일 20,000건 제한** — 200종목 × 연 1회면 여유 |
| **OpenDART `fnlttSinglAcntAll`** (전체 재무제표) | 자본총계(지배·x3 분모), 당기순이익, 영업활동현금흐름·유형자산취득(→FCF·x51) | `OpenDartReader.finstate_all(ticker, year)` CFS 우선 | 상동 |
| **OpenDART `corpCode.xml`** | 티커 ↔ corp_code 매핑 | zip 1회 다운로드 → 로컬 캐시 | 1회성 |
| **pykrx** | KOSPI200 구성종목, 시세 기준 **DIV(배당수익률)·DPS·EPS·BPS·PER·PBR** 일별 스냅샷 | `stock.get_index_portfolio_deposit_file("1028")`, `stock.get_market_fundamental_by_ticker(date)` | 무료 (KRX 스크래핑 — 요청 간격 유지) |
| **FnGuide 스냅샷** (comp.fnguide.com) | 배당성향·ROE·DPS 5개년 시계열 — **교차검증·보조** | 종목별 Snapshot 페이지 크롤링 (네이버 수집기와 동일 패턴) | 무료, 크롤링 예의 필요 |
| **네이버 금융** | 배당수익률·DPS 현재값 (간이 검증) | 기존 크롤러 재사용 | 무료 |

**우선순위 원칙**: 공시 원본(OpenDART) = 정본, pykrx = 시세성 지표 정본,
FnGuide/네이버 = 교차검증용. 두 소스가 5% 이상 어긋나면 `data_quality`에 기록하고 DART 값 채택.

### 3.2 지표 → 소스 매핑표

| 산식 항목 | 필요한 원천값 | 1순위 소스 (필드) | 보조/검증 |
|---|---|---|---|
| x11, x12 | 연도별 배당 지급 여부, 흑자 여부 | alotMatter (주당 현금배당금, 당기순이익 — 3개년/호출) | pykrx DPS, FnGuide |
| x21, x22 | **배당성향** 시계열, 순이익 변동 | alotMatter **(연결)현금배당성향(%)** — 직접 제공 | 직접 계산: 현금배당금총액 ÷ 지배주주순이익 (fnlttSinglAcntAll) — 적자·왜곡 연도 재계산용 |
| x3 | 현금배당금총액, 자기자본 | alotMatter(현금배당금총액) + fnlttSinglAcntAll(자본총계·지배) | BPS×상장주식수(pykrx) 근사 |
| x41 | 배당성향 (유니버스 전체) | 위 x21과 동일 → 200종목 일괄 수집 후 백분위 | — |
| x42 | ROE | 계산: 당기순이익 ÷ 평균자기자본 (fnlttSinglAcntAll 2개년) | FnGuide Snapshot ROE |

### 3.3 수집 파이프라인 설계 (`s8_dividend_score.py` — 차기 구현)

```
[연 1회 배치 + 수시 재실행 (resume)]
1. pykrx: KOSPI200 구성종목 스냅샷 → universe_snapshot (기준일 포함 저장 — 선견편향 방지)
2. corpCode.xml 캐시 → 200종목 corp_code 매핑
3. 종목 루프 (200):
   a. alotMatter(t년) → 3개년 배당 블록  ← v2용은 t-2, t-4년도 추가 호출 (10개년)
   b. fnlttSinglAcntAll(t, t-1, …) → 자본총계·순이익·현금흐름
   c. dividend_facts upsert (ticker × fiscal_year, 원시값 + source + fetched_at)
4. 전 종목 수집 완료 후(백분위는 전체가 있어야 계산 가능):
   a. v1 스코어 → dividend_scores upsert (score_version='v1')
   b. v2 스코어 → 동일 (score_version='v2')
5. 종목별 embed_text 생성(§4.3) → BGE-M3 임베딩 → embedding 컬럼 upsert
6. 검산 리포트: GP 분포 히스토그램, A/B/C 비중, 결측·flags 집계 → eval 대시보드에 1장 추가
```

- 호출량 추정: alotMatter 200×(1~5회) + finstate_all 200×(2~5회) ≈ **2,000~3,000건/연**
  → DART 일 한도(20,000)의 15% 이내. 재시도·캐시(로컬 JSON, `fetched_at` TTL 30일) 포함.
- 갱신 시점: **매년 4월 초** (12월 결산 사업보고서 마감 3/31 직후). 스케줄러 등록 전에는 수동 bat.
- 기존 `common.py`의 `upsert()`/`done_ids()` resume 패턴 그대로 재사용.

## 4. RAG DB 적재 설계

### 4.1 테이블 (DDL은 `002_dividend_score.sql`)

```
dividend_facts   ticker × fiscal_year — 원시값 (DPS, 배당성향, 순이익, 자기자본, FCF, DIV …)
                 + source, data_quality(jsonb), fetched_at        ← 재계산 가능한 정본
dividend_scores  ticker × fiscal_year × score_version — x11…x53 항목별 점수 + GP,
                 score100, grp(A/B/C), rank/pct_in_universe, flags, details(jsonb),
                 embed_text, embedding halfvec(1024)              ← 검색·인용 대상
```

- `companies(ticker)` FK로 기존 엔티티 링킹(doc_companies)과 자연 조인 —
  "리포트 검색 결과 종목의 배당 스코어"를 즉시 붙일 수 있다.
- 항목별 점수를 개별 컬럼으로 두는 이유: "x22 페널티 받은 종목" 같은 조건 검색과
  회귀 분석(어떤 항목이 수익률을 설명하나)을 SQL로 바로 하기 위함.
- `details` JSONB에는 채점 근거 원시값 스냅샷(Δ배당성향, 백분위 등)을 넣어
  **점수의 감사추적(audit trail)** 을 보장 — RAG 답변이 "왜 A그룹인가"까지 인용 가능.

### 4.2 검색 흐름 통합

```
질문 "배당 늘리는 반도체 대형주 뭐 있어?"
 → (기존) 하이브리드 검색: 리포트 청크·이미지
 → (추가) match_dividend_scores RPC: embed_text 벡터 검색 + grp/score 필터
 → s7_query 프롬프트에 [배당 스코어 카드] 블록 추가 (점수·근거·연도 명시)
```

- 티커가 식별된 질문은 벡터 검색 없이 `ticker = ?` 직조회 (정형 데이터가 정확).
- 정량 조건("배당성향 30% 이하", "A그룹만")은 SQL 필터 → 벡터 검색과 RRF 결합.

### 4.3 embed_text 생성 규칙 (RAG 검색 품질의 핵심)

숫자 테이블은 그대로 임베딩하면 검색이 안 된다. **채점 결과를 한국어 서술문으로 풀어** 임베딩:

```
템플릿:
"{종목명}({티커}) {사업연도} 배당 스코어 {GP:+.1f}점, {grp}그룹
 (KOSPI200 {pct}위/상위 {top_pct}%). 과거 3년 배당 {지급횟수}회 지급.
 배당성향 {payout:.1f}%로 전년 대비 {증감서술}. 현금배당률 추세 {추세서술}.
 ROE {roe:.1f}%로 유니버스 {roe_위치}. {v2추가: FCF 배당커버리지 {cov:.1f}배,
 {연속지급}년 연속 지급}. {flags 서술: 무리한 배당 경고 / 특별배당 포함 / 적자연도 등}"

예:
"삼성전자(005930) 2025 사업연도 배당 스코어 +2.3점, A그룹 (KOSPI200 24위/상위 12%).
 과거 3년 배당 3회 지급. 배당성향 27.8%로 전년 대비 2.1%p 상승. 현금배당률 추세
 3개년 연속 상승. ROE 11.2%로 유니버스 상위 35%. FCF 배당커버리지 2.4배,
 9년 연속 지급."
```

- 문장에 **종목명·티커·연도·그룹·핵심 수치**가 모두 들어가야 dense/BM25 양쪽에서 잡힌다.
- flags는 반드시 서술에 포함 (경고를 숨기면 RAG 답변이 왜곡됨).

| RAG 검색 | 골든 질문 5개 추가 ("배당 좋은 종목", "A그룹 반도체" 등) | hit@5에 스코어 카드 포함 |
